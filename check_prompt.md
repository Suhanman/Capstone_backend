"이제 내 담당 역할인 '코어 & 보안 도메인 (User & Auth)' 코드를 집중적으로 점검하고 로직을 완벽히 이해하려고 해. 다른 도메인은 제외하고, 내가 담당하는 [JWT 로그인/회원가입, 구글 메일/캘린더 연동, 구글 Pub/Sub Push 메일 수신, 관리자의 회원 관리] 기능에 대해서만 아래 4가지를 작성해 줘.

1. 핵심 로직 흐름 설명 (데이터 파이프라인):

서비스 자체 JWT 기반 인증 흐름과 Google OAuth(Access/Refresh Token) 연동 흐름이 각각 어떤 클래스(Controller -> Service -> Repository)를 거쳐 DB에 저장되는지, 데이터의 흐름을 내가 다른 사람에게 설명할 수 있을 정도로 쉽게 요약해 줘.

특히 구글 메일 수신을 위한 'Google Pub/Sub Push' 로직이 현재 어떤 구조로 외부 요청을 받아 RabbitMQ 등으로 넘기도록 설계되어 있는지 구체적인 흐름을 알려줘.

2. 권한(Role) 제어 및 보안 점검:

일반 사용자용 API와 관리자용 API의 권한 분리가 Spring Security 필터 체인 레벨에서 어떻게 구현되어 있는지, 혹시 권한 탈취 취약점은 없는지 점검해 줘.

3. claude.md 규칙 준수 여부 검증:

구글 API 연동 시 순수 HTTP 요청이 아닌 공식 Java 클라이언트 라이브러리(google-api-client 등)를 제대로 사용했는지 확인해 줘.

상태 유지(State)를 위해 임시 JWT를 발급하는 규칙과 필수 Scope 누락 검증 로직이 완벽하게 들어갔는지 확인해 줘.

4. 미구현 및 시급한 보완점:

내 담당 영역 중 아직 껍데기만 있거나, 핵심 로직이 빠져서 당장 내가 클로드에게 지시해서 완성해야 하는 가장 시급한 태스크 3가지를 꼽아줘."
